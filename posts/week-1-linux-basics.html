2) Deploying Nginx + Static React App

Goal: Serve a built React app using Nginx on an Ubuntu VM.

  1. Install Nginx 
    $ sudo apt update
$ sudo apt install -y nginx
  2. Allow HTTP (and HTTPS) through the firewall (optional if UFW enabled) 
    $ sudo ufw allow 'Nginx Full'
$ sudo ufw status
  3. Build your React app locally (or on the VM) 
    $ npm run build
# outputs: build/ (static assets)
  4. Copy build to Nginx web root 
    $ sudo rm -rf /var/www/html/*
$ sudo cp -r build/* /var/www/html/
  5. Check Nginx status & reload on config changes 
    $ systemctl status nginx
$ sudo systemctl reload nginx
  6. Validate from the server 
    $ curl -I http://localhost
HTTP/1.1 200 OK

Tip: For custom domains, create an A record pointing to your VM‚Äôs public IP, then add a server block in /etc/nginx/sites-available/ and symlink to sites-enabled. Use sudo nginx -t to test config.

3) Health & Verification Checks

  • sudo netstat -tulnp ‚Üí Ensure :80 or :443 is listening.
  • ps aux | grep nginx ‚Üí Worker/master processes present.
  • tail -f /var/log/nginx/access.log ‚Üí Requests hitting your server.
  • tail -f /var/log/nginx/error.log ‚Üí Misconfig & 4xx/5xx clues.
  • df -h / free -h ‚Üí Resource headroom ok.

4) Key Takeaways

  • Linux commands = visibility; Nginx = delivery. Together, they de-risk deployments.
  • Always verify with ports, processes, logs, and simple curl checks.
  • Keep configs minimal at first; add TLS, caching, and hardening as next steps.
